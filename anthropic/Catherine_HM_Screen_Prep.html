Question 1: Deal Deep Dive

Things to Avoid

1/ Don't talk for more than 2.5 minutes on any answer — McKinsey-trained interviewers value conciseness; she'll mentally check out. Give the headline and let her pull detail with follow-ups.
2/ Don't open with "Great question" — it's filler. She knows it's a good question. Just answer.
3/ Avoid the words "just", "I think", "to be honest", "basically" — they minimise you. Use instead: "In my experience", "What I've found", "To be direct."
4/ Don't over-explain Google context — she was at Google. She knows how it works. Reference briefly and move to the result.
5/ Don't trash OpenAI — it's unprofessional. If it comes up: "Enterprises I've spoken to have concerns about governance and data practices."
6/ Don't say "I love AI" or "AI is the future" — vague enthusiasm, she's heard it from every candidate. Be specific: "I built multi-agent systems for credit analysis in production."
7/ Don't volunteer Ringkas problems — she might wonder if you're running away. Frame as: "I built what I set out to build."
8/ Don't ask about comp, title or equity — not at this stage; wait for the recruiter or offer stage.
9/ Don't try to fill silence — she may pause to think or take notes. Silence is fine.
10/ Don't mention "country manager" ambition — too much too soon. Focus on the AE role. If asked about growth: "I want to build something meaningful. The path will follow."

Product Knowledge: How Anthropic Makes Money

Don't volunteer this section during the interview — but if she asks about consumption, pricing or tokenisation, you need to sound fluent. This maps directly to the EDP/consumption model you already sold at AWS.

Tokens

A token is a chunk of text; 1 word is approximately 1.3 tokens. 1 million tokens equals roughly 750,000 words; a 10-page document is approximately 3,500 tokens. Anthropic charges per million tokens, separately for input (what you send to Claude) and output (what Claude generates back). Output is always more expensive because the model is generating, not just reading.

Pricing

Current Claude 4.5 series:

1/ Haiku 4.5 at $1 input / $5 output per million tokens (fastest, use for simple tasks, routing, classification)
2/ Sonnet 4.5 at $3 input / $15 output per million tokens (balanced — this is what most enterprise work runs on)
3/ Opus 4.5 at $5 input / $25 output per million tokens (most capable — complex reasoning, high-stakes use cases)

How Enterprises Buy Claude

There are three main channels:

1/ API direct, where a dev team gets an API key and pays as they go — fast to start, good for POCs
2/ Claude Enterprise, which is an org-wide platform with annual contract, SSO, admin controls — predictable cost and governance
3/ Cloud marketplace (AWS Bedrock or Google Vertex AI), where Claude is billed through the enterprise's existing AWS or GCP spend — no new procurement cycle required

The Bedrock/Vertex angle is critical for Industries AE. A bank already spending $10M/yr on AWS can allocate part of that committed spend to Claude without a new vendor approval process. This removes procurement friction entirely, which is a major advantage in regulated industries where new vendor onboarding can take 6-12 months.

Cost Levers

There are three main levers for enterprise customers to manage costs:

1/ Prompt caching — if the same large context is sent repeatedly (i.e, a 100-page compliance manual as a system prompt), cached reads cost 90% less after the first write
2/ Batch API — for non-real-time workloads such as overnight document processing, there is a 50% discount on both input and output tokens
3/ Model routing — a well-architected system routes 80% of tasks to Haiku ($1/MTok input) and reserves Opus ($5/MTok input) for the 20% that require complex reasoning. This can reduce overall costs by 60-70%

Maps to AWS

The pricing model maps directly to what you sold at AWS:

1/ on-demand compute = pay-as-you-go API
2/ reserved instances = annual committed spend
3/ EDP = enterprise agreement with volume discounts
4/ spot instances = Batch API (50% off for non-real-time)
5/ CloudFront caching = prompt caching (90% off for repeated context)

Sales Motion

The consumption growth motion in one line: land with a bounded POC ($5-20K/mo) in a single use case, prove ROI, expand to additional use cases and teams, convert to annual committed spend agreement, then grow token consumption as the customer embeds Claude deeper into their workflows. This is the same EDP motion you sold at AWS — customer commits to $X/year, gets a discount, Anthropic gets revenue predictability. Tokens are the new compute hours.
